#  05. Data Types

## 5.1 Dynamic Typing

- 자바스크립트는 느슨한 타입(loosely typed), 동적(dynamic) 언어이다.
- 타입은 프로그램이 처리되는 과정에서 자동으로 파악된다.

```javascript
var foo = 'Hello';	// foo는 String 타입을 가진다.
var foo = 1;		// foo는 Number 타입을 가진다.
```



## 5.2 Data Types

- ECMAScript의 데이터 타입은 다음과 같이 나누어 볼 수 있다. 
  - [데이터(data)와 구조(structure)](https://developer.mozilla.org/ko/docs/Web/JavaScript/Data_structures#%EB%8D%B0%EC%9D%B4%ED%84%B0_%EB%B0%8F_%EA%B5%AC%EC%A1%B0_%EC%9C%A0%ED%98%95)
  - 원시(primitive)와 객체(object)



## 5.3 Primitive Value

- 원시 값(primitive value) 혹은 원시 데이터 타입(primitive data type)은 immutable하다.
- 원시 데이터 타입에는 6가지 타입이 있다. 이중 `undefined`와 `null`를 제외한 모든 원시 값은 원시 래퍼 객체(wrapper)를 가진다.
  - Undefined Type
  - Null Type
  - Boolean Type
  - String Type
  - Numeric Type
  - Symbol Type(ECMAScript 6에 도입)



### 5.3.1 Undefined Type

```javascript
var x;
```

- `undefined`는 **전역 객체의 프로퍼티로, 전역 범위를 가진 변수**이다.
- `undefined`는 전역 변수로서 식별자이나, 예약어는 아니다.
- `undefined` 변수의 초기 값은 `undefined` 원시 값이다.
- 값을 할당하지 않은 변수는 값으로 `undefined`를 가진다.
- 함수는 반환값을 지정하지 않으면 `undefined`를 반환한다.
- 인수에 값을 전달하지 않으면 값으로 `undefined`가 할당된다.

```javascript
var x;
if (typeof x === 'undefined') {
    console.log('type of x is undefined')
}

if (x === undefined) {
    console.log('value of x is undefined')
}
```



### 5.3.2 Null Type

```javascript
var x = null;
```

- `null`은 **변수가 선언되었으나 어떤 객체도 참조하고 있지 않음**을 가리킨다.
- `null`은 리터럴로 primitive value에 속하나, 그 타입은 `object`이다.

```javascript
typeof null === 'object'	// true
```



### 5.3.3 Boolean Type

```javascript
var x = false;
if (x) {
    // 실행되지 않는다.
}
```

- `boolean` 타입은 논리 연산의 결과로서 `true`와 `false`만을 그 값으로 가진다.



#### 5.3.3.1 Boolean 래퍼 객체

| 함수            | 반환      |
| --------------- | --------- |
| `Boolean()`     | `boolean` |
| `new Boolean()` | `object`  |

- `boolean` 타입의 원시 값은 `Boolean` 래퍼 객체를 가진다.

```javascript
var x = new Boolean(false);
if (x) {
    // 실행된다.
}
```

- 불리언이 아닌 값을 변환할 때에는`Boolean` 객체가 아니라 `Boolean` 함수를 사용한다.

```javascript
var x = new Boolean(expression);	// Boolean 생성자를 이용하여 Boolean 객체 반환
var x = Boolean(expression);		// Boolean 원시 값 반환
```

- 인자가 falsy라면, `Boolean` 객체는 `false` 값으로 초기화된다.
- 인자가 truthy라면, `Boolean` 객체는 `true` 값으로 초기화된다.

```javascript
var a = new Boolean();			// false 값으로 초기화
var b = new Boolean(false);		// false 값으로 초기화
var c = new Boolean(true);		// true 값으로 초기화
var d = new Boolean('false');	// true 값으로 초기화
```



### 5.3.4 Numeric Type

- numeric type에는 `number`와 `bigint`가 있다.



#### 5.3.4.1 Number Type

```javascript
typeof 1 === 'number'	// true
typeof 1.23 === 'number'	// true
```

```js
let a = 1;
```

- `number` 타입은 정수와 부동소수점 숫자를 표현한다.
- `number`타입은 special numeric value를 가진다.
  - `+Infinity` 혹은 `Infinity`:  무한대
  - `-Infinity`
  - `NaN`: **N**ot **a** **N**umber

```js
1 / +0	// Infinity
1 / -0	// -Infinity
```



##### 5.3.4.1.1 NaN

- `NaN`은 **전역 객체의 프로퍼티로, 전역 범위를 가진 변수**이다.
- `NaN` 변수의 초기값은 `Number.NaN`과 같다.

```js
isNaN(NaN);         // true
isNaN(Number.NaN);  // true
```

- `isNaN`과 `Number.isNaN`은 그 의미가 다르다.
  - `isNan`: 현재 값 혹은 변환했을 때의 값이 `NaN`이어야지만 `true`를 반환
  - `Number.isNaN`: 현재 값이 `NaN`이어야지만 `true`를 반환

```js
isNaN('a');			// true
Number.isNaN('a')	// false
```



##### 5.3.4.1.2 Number 래퍼 객체

| 함수           | 반환     |
| -------------- | -------- |
| `Number()`     | `number` |
| `new Number()` | `object` |

- `number` 타입의 원시 값은 `Number` 래퍼 객체를 가진다.

```js
var a = new Number('123');		// (a === 123)은 false
var b = Number('123');			// (b === 123)은 true
a instanceof Number; 			// true
b instanceof Number; 		// false
typeof a === 'object';		// true
typeof b === 'number';		// true
```



#### 5.3.4.2 BigInt Type

```js
const bigInt = 1234567890123456789012345678901234567890n;
typeof bigInt === 'bigint';		// true
```

- `bigint` 타입은 아주 큰 숫자나 높은 정밀도를 표현할 때 사용한다.
- `bigint` 타입의 값은 정수 리터럴 끝에 `n`을 붙여 만든다.



##### 5.3.4.2.1 BigInt 래퍼 객체

| 함수           | 반환     |
| -------------- | -------- |
| `BigInt()`     | `bigint` |
| `new BigInt()` | `object` |

- `bigint` 타입의 원시 값은 `BigInt` 래퍼 객체를 가진다.

```js
const bigInt = BigInt(1234567890123456789012345678901234567890n);
```



### 5.3.5 String Type

```js
const NAME = 'umi';
```

- `string` 타입의 값은 `''`이나 `""` 혹은 ``  ` `  ``로 표현한다.
- 인덱스나 `charAt` 메서드로 문자에 접근할 수 있다.

```js
'umi'[0];		// 'u'
'umi'.charAt(0)	// 'u'
```



#### 5.3.5.1 String 래퍼 객체

| 함수           | 반환     |
| -------------- | -------- |
| `String()`     | `string` |
| `new String()` | `object` |

- `string` 타입의 원시 값은 `String` 래퍼 객체를 가진다.
- `String()`은 원시 스트링(primitive string)을 반환한다.



#### 5.3.5.2 템플릿 리터럴

- ECMAScript 2015 이후 도입
- 문자열을 `` ` ``(백틱)으로 감싸고, 변수나 표현식을 `${}`으로 감싸 문자열 안에 넣는다.

```js
const NAME = 'umi';
console.log(`캐릭터 이름은 ${NAME}!`);
```



### 5.3.6 Symbol Type

```js
var foo = Symbol('foo');
Symbol('foo') === Symbol('foo');		// false
Symbol('foo') === 'foo';				// false
Symbol('foo').toString();				// Symbol(foo)
```

- ECMAScript 6에 도입
- `symbol` 타입의 값은 유일하고 변경 불가능한 원시 값이다(A Symbol is a **unique** and **immutable** primitive value)
- `symbol` 타입의 값은 객체 프로퍼티의 `key` 값(식별자)으로 사용된다.



#### 5.3.6.1 Symbol 래퍼 객체

| 함수       | 반환     |
| ---------- | -------- |
| `Symbol()` | `symbol` |

- `symbol` 타입의 원시 값은 `Symbol` 래퍼 객체를 가진다.
- `new Symbol()`을 지원하지 않아 `TypeError`가 발생한다. `Symbol` 래퍼 객체를 생성하고 싶다면 다음과 같이 한다.

```js
var foo = Symbol('foo');
var obj = Object(foo);
```



## 5.4 Structure

- 타입은 데이터와 구조(혹은 객체)로 나누어 볼 수 있다.
  - 데이터: `null` 값을 제외한 모든 primitive value
  - 구조: 객체, 함수, `null`

```js
typeof new Object() === 'object';
typeof function(){} === 'function';
typeof null === 'object';
```

- 객체는 데이터 구조이다.
- 함수는 비데이터 구조이다.
  - 함수는 객체이지만, `typeof`의 연산 결과는 `function`이다.
- `null`은 [Structural Root Primitive](https://developer.mozilla.org/ko/docs/Web/JavaScript/Data_structures#%EA%B5%AC%EC%A1%B0_%EC%9C%A0%ED%98%95)이다.
  - `null`은 객체가 아닌 primitive value이나, `typeof`의 연산 결과는 `object`이다.



## 5.5 Objects

- 객체는 프로퍼티를 가지는 collection으로, 복잡한 데이터 구조를 표현할 때 사용한다.
- 프로퍼티는 키와 값의 매핑이다.
  - 프로퍼티의 키는 `String` 혹은 `Symbol` 값이다. 
  - 프로퍼티의 값은 어떠한 자료형이든 가능하다.



## 5.6 typeof operator

```javascript
typeof x;
typeof(x);
```

- `tyoeof` 연산자는 단항 연산자이다.
- 피연산자의 타입을 `string`으로 반환한다.
